# Indicadores para análisis competitivo

> Framework de medición sistemática para analizar sitios web del rubro Seguridad e Higiene.
> Cada sitio analizado debería responder a TODOS los puntos de este checklist.

---

## Cómo usar este documento

1. **Por cada sitio a analizar**, crear un archivo en `researching/sitios/<nombre-sitio>.md` que responda este checklist completo.
2. **Una vez analizados 5-8 sitios**, sintetizar en `researching/sintesis.md`: patrones comunes, oportunidades, decisiones para el sitio de Lucas.
3. **Cada indicador tiene un "Por qué importa"** — para entender qué decisión va a depender de ese dato y no perder tiempo midiendo cosas irrelevantes.

---

## 1. Estructura y arquitectura de información

| Indicador | Qué medir |
|---|---|
| **Cantidad total de páginas** | Contar páginas únicas (excluir variantes de query string) |
| **Tipos de páginas** | Home / Sobre / Servicios general / Servicios individuales / Capacitación / Clientes / Contacto / Blog / Recursos / Legal |
| **Páginas individuales por servicio** | ¿Cada servicio tiene su URL propia o todo está en una sola página? |
| **Profundidad de navegación** | ¿Cuántos clicks desde Home a la info más profunda? |
| **Menú principal** | Cantidad de items, jerarquía (¿hay submenús/dropdowns?), orden |
| **Footer** | ¿Replica navegación? ¿Tiene info de contacto? ¿Newsletter? ¿Redes? |

**Por qué importa**: define la estructura mínima viable para el sitio de Lucas y revela si los líderes apuestan por SEO de páginas individuales o todo-en-uno.

---

## 2. Hero / above the fold

| Indicador | Qué medir |
|---|---|
| **Tipo de hero** | Foto profesional / foto de obra / ilustración / video / plano / split |
| **Headline principal** | Texto exacto, cantidad de palabras, tono |
| **Subheadline** | Texto exacto, cuánto explica vs cuánto vende |
| **CTA principal** | Texto del botón, color, ubicación (¿cerca del headline o flotante?) |
| **CTA secundario** | ¿Existe? ¿A qué dirige? |
| **Foto de la persona** | ¿Aparece el profesional? ¿O solo logo + texto? |
| **Datos de contacto visibles ATF** | ¿Mail/teléfono/WhatsApp en pantalla sin scrollear? |

**Por qué importa**: el hero es lo único que el 90% de los visitantes va a ver. Si Lucas quiere transmitir "soy yo, profesional individual", el hero tiene que mostrar SU cara y SU nombre — no un logo abstracto.

---

## 3. Tono y voz

| Indicador | Qué medir |
|---|---|
| **Persona gramatical** | 1ra singular ("yo") / 1ra plural ("nosotros") / 3ra ("la consultora") |
| **Voseo vs tuteo vs usted** | ¿Cómo le habla al lector? |
| **Formalidad** | Formal corporativo / profesional cercano / casual |
| **Densidad técnica** | Cuánta jerga (Ley 19587, Decreto 351/79, IRAM, etc.) en la home |
| **Largo de frases** | Frases cortas y punzantes vs párrafos largos descriptivos |
| **Verbos típicos** | "Ofrecemos", "brindamos", "te acompaño", "evaluamos", "garantizamos" |
| **Emoción vs racionalidad** | ¿Apela a tranquilidad, miedo, eficiencia, cumplimiento? |

**Por qué importa**: define el registro del copy de Lucas. Si todos hablan en "nosotros corporativo", él puede diferenciarse en primera persona singular ("Soy Lucas, te acompaño...").

---

## 4. Servicios — cómo se presentan

| Indicador | Qué medir |
|---|---|
| **Formato de presentación** | Lista / cards / tabs / acordeón / timeline |
| **Cantidad de servicios listados** | Total y agrupados (servicios principales vs sub-servicios) |
| **Líneas de copy por servicio** | Cuánto detalle se da: 1 línea, 1 párrafo, 1 sección entera |
| **Iconos / imágenes por servicio** | ¿Hay iconografía? ¿Estilo flat / outline / detallado? |
| **Sub-servicios visibles** | ¿Muestran el desglose o solo títulos genéricos? |
| **Precios mostrados** | Sí / No / "Pedí presupuesto" |
| **CTA por servicio** | ¿Cada servicio tiene su propio CTA? |

**Por qué importa**: Lucas tiene 3 áreas (Habilitaciones, Mediciones, Incendios) con muchos sub-servicios. Hay que decidir si los muestra todos o filtra para no abrumar.

---

## 5. Pruebas sociales y autoridad

| Indicador | Qué medir |
|---|---|
| **Logos de clientes** | Cantidad mostrada, calidad de los logos (real / generic) |
| **Testimonios** | Cantidad, formato (texto / video / con foto) |
| **Casos de éxito** | ¿Hay portfolio? Detalle: foto + descripción + resultado |
| **Números de autoridad** | "+X años", "+X clientes", "+X obras supervisadas" |
| **Certificaciones / matrícula** | ¿Se muestran? ¿Dónde? (footer / hero / sobre) |
| **Asociaciones profesionales** | COPIME / cámaras / colegios profesionales |
| **Premios o menciones** | Si existen |

**Por qué importa**: Lucas tiene matrícula profesional y eso es un activo. Hay que decidir cómo exhibirlo (sello en footer vs destacado en hero).

---

## 6. Contacto y conversión

| Indicador | Qué medir |
|---|---|
| **Botón flotante de WhatsApp** | ¿Existe? ¿En todas las páginas? Tamaño/visibilidad |
| **Formulario de contacto** | ¿Existe? Cantidad de campos, fricción de uso |
| **Mail visible** | Sí / No / dónde |
| **Teléfono visible** | Sí / No / formato (clickable en mobile?) |
| **Mapa de ubicación** | Embed de Google Maps? |
| **Horario de atención** | ¿Se declara? |
| **Tiempo de respuesta declarado** | "Respondo en 24hs", "atención inmediata", etc. |
| **Click-to-call mobile** | ¿El número es clickable en móvil? |

**Por qué importa**: la pregunta crítica es si el sitio convierte por WhatsApp (modelo Lucas) o por formulario (modelo corporativo).

---

## 7. Diseño visual

| Indicador | Qué medir |
|---|---|
| **Paleta primaria** | Color de marca + acento (3-4 colores principales con HEX si se puede) |
| **Tipografía** | Serif / sans / display. Nombre si se identifica |
| **Tamaño de tipografía base** | Aprox px del cuerpo (chico / medio / grande) |
| **Tipo de imagery** | Fotos reales / stock / ilustraciones / 3D / mixto |
| **Iconografía** | Outline / filled / detallado / sin iconos |
| **Densidad visual** | Denso (mucho por sección) / aireado (mucho whitespace) |
| **Componentes destacados** | Tabs / accordion / slider / parallax / animaciones |
| **Estilo general** | Industrial-clásico / SaaS-moderno / corporativo-formal / artesanal |
| **Mobile responsive** | Sí / No / mediocre |

**Por qué importa**: define qué aspecto debería tener el sitio de Lucas para encajar en su rubro. Salirse del patrón visual del nicho puede ser positivo (diferenciación) o negativo (no inspira confianza).

---

## 8. SEO observable

| Indicador | Qué medir |
|---|---|
| **Title pattern** | Estructura del `<title>` (ej: "Servicio - Marca - Ciudad") |
| **H1 de la home** | Texto exacto |
| **Estructura de URLs** | Limpia (`/servicios/habilitaciones`) o con queries (`?id=123`) |
| **Schema.org visible** | LocalBusiness / ProfessionalService / Service / Person |
| **Mención explícita de keywords locales** | "CABA", "Buenos Aires", "habilitaciones CABA" |
| **Mención de leyes/normas** | Ley 19587, Ley 5920, Decreto 351/79 |
| **Velocidad de carga aparente** | Bien / regular / lento |
| **Sitemap accesible** | `/sitemap.xml` |
| **Robots.txt** | `/robots.txt` |

**Por qué importa**: Lucas necesita rankear en CABA. Hay que ver qué keywords están ocupando los líderes y dónde hay huecos.

---

## 9. Contenido extra y autoridad de marca

| Indicador | Qué medir |
|---|---|
| **Sección "Por qué elegirnos"** | ¿Existe? Cantidad de bullets, tono |
| **Sección "Cómo trabajamos"** | ¿Hay un proceso paso-a-paso visible? |
| **FAQ** | ¿Existe? Cuántas preguntas |
| **Blog / artículos** | ¿Existe? Cantidad de posts, frecuencia, calidad |
| **Recursos descargables** | Guías, checklists, modelos de protocolos |
| **Video institucional** | ¿Existe? Dónde, qué muestra |
| **Página "Sobre"** | ¿Hay foto + bio del responsable? Largo de la bio |

**Por qué importa**: estas secciones son las que generan tiempo de permanencia y autoridad para SEO. Lucas puede agregarlas progresivamente.

---

## 10. Particularidades del rubro SyH

| Indicador | Qué medir |
|---|---|
| **Mención de Ley 5920 (CABA)** | ¿La nombran? ¿Tiene página dedicada? |
| **Mención de Ley 19.587 + Dto 351/79 (nacional)** | Igual |
| **Foco geográfico** | CABA / GBA / nacional / internacional |
| **Tipos de empresa target** | PyMEs / grandes / comercios / industria / construcción / oficinas |
| **Sectores específicos** | Industrial / gastronomía / hotelería / educación / salud |
| **Servicios "core"** | ¿Cuáles repiten más en su comunicación? |
| **Mención de matrícula del profesional firmante** | ¿Aparece número de matrícula visible? |
| **Mención de equipamiento propio** | "Instrumental calibrado", "laboratorio propio", etc. |
| **Acreditaciones de organismos** | AGC (CABA), SRT, organismos provinciales |

**Por qué importa**: el rubro tiene reglas y convenciones que el cliente final espera ver. Si Lucas no menciona Ley 5920 en su sitio, va a parecer que no domina el marco regulatorio CABA.

## Síntesis final (a completar después de los análisis)

Cuando estén los 5-7 análisis individuales, el documento `researching/sintesis.md` debería responder:

1. ¿Qué patrón de **estructura de sitio** se repite y debería seguir Lucas?
2. ¿Qué **tono y persona gramatical** usar para diferenciarse?
3. ¿Qué **diferenciales** están saturados y cuáles tienen hueco?
4. ¿Qué **paleta y diseño** valida el rubro?
5. ¿Qué **5 cosas concretas** robar y qué **5 cosas concretas** evitar?
